# 벤치마크 — 필터 오버헤드 & Python vs JVM 처리량

PII 탐지/마스킹 **핫패스**(정규화 → L1 정규식·체크섬 → 인젝션 점검 → 마스킹)가 요청당 얹는
지연과, 단일/멀티코어 처리량을 잰다. 업스트림 LLM 호출(초 단위)이 전체 지연을 지배하므로
여기서 재는 값은 **"필터가 더하는 밀리초 예산"**과 **"코어당·머신당 처리 상한"**이다.

- Python baseline: [`bench_py.py`](bench_py.py) — `./.venv/bin/python bench/bench_py.py`
- JVM 비교: [`Bench.java`](Bench.java) — `java bench/Bench.java` (JDK 21+, 빌드 도구 불필요)
- 동일 워크로드(같은 4개 문자열), 동일 작업(정규화+L1+인젝션+마스킹).

## 결과 (18-core dev machine, 단일 측정 · 지표는 참고치)

### 단일 스레드 지연 (µs, heavy = PII 4종 포함 문장)

| payload | Python p50 | Python p99 | Java p50 | Java p99 |
|---|---|---|---|---|
| clean (PII 없음) | 19.1 | 30.2 | 15.0 | 33.8 |
| light (전화 1건) | 13.3 | 15.3 | 8.0 | 13.8 |
| heavy (전화+이메일+주민+카드) | 46.5 | 52.0 | 35.2 | 47.8 |
| injection | 12.5 | 13.6 | 7.7 | 11.7 |

→ **필터 오버헤드는 최악(heavy)에도 p99 ~52µs.** LLM 호출(수백~수천 ms) 대비 무시할 수준.
→ 단일 스레드 per-op 은 Java 가 ~1.3–1.6× 빠름(정규식 엔진 + JIT).

### 처리량 (heavy payload, ops/s)

| | Python | Java | 비고 |
|---|---|---|---|
| 1 스레드 | ~20,700 | ~27,500 | Java 1.3× |
| 18 스레드 | **~20,700 (평평)** | **~362,000** | **Java ~17×** |

→ **핵심**: Python 은 GIL 로 CPU 작업이 스레드 수와 무관하게 평평(1코어). 확장하려면
`multiprocessing`(프로세스당 메모리 + IPC 오버헤드). JVM 플랫폼 스레드는 코어에 ~선형 확장.

### 단계 분해 (Python, heavy, p50)

| 단계 | µs | 비중 |
|---|---|---|
| normalize (NFKC+제거) | 0.9 | 저렴 |
| detect_l1 + merge | 25.6 | 지배적(정규식·체크섬) |
| detect_injection | 18.0 | 큼(카테고리 패턴 다수) |

## 결론

- **오버헤드는 문제 아님**: 필터는 요청당 수십 µs. 게이트웨이 병목은 항상 업스트림 LLM.
- **처리량/밀도**: 동일 트래픽을 Python 은 코어 수만큼 **프로세스**로 띄워야 하고(메모리↑),
  JVM 은 **스레드**로 한 프로세스 안에서 코어를 채운다 → 인스턴스 밀도·운영 단순성에서 JVM 유리.
- 그래서 "Python(LiteLLM)으로 먼저 만들고, 처리량이 중요해지면 JVM(가상 스레드) 포트" 라는
  프로젝트 방향이 수치로 뒷받침된다(DESIGN §10.1).

## 정직한 한계 (측정 해석 시 주의)

- **단일 머신·단일 측정**. 절대값은 하드웨어·부하·JIT 상태에 따라 흔들린다. 상대 경향만 보라.
- **`Bench.java` 는 핫패스의 대표 재구현**이지 사양 100% 패리티가 아니다(같은 입력·같은 종류의
  작업을 재는 것이 목적). 정규식 엔진(`java.util.regex` vs `re`)·문자열 처리 차이가 섞여 있다.
- **CPU 마이크로벤치**다. **가상 스레드**의 진짜 이점은 여기(CPU 바운드)가 아니라 프록시의
  **I/O 동시성**(업스트림 대기 중 수천 요청 보류)에 있다 — 이는 실게이트웨이 부하시험의 몫.
- 페이로드는 합성이며 실제 트래픽 분포를 대표하지 않는다.
